# SA/BA System Prompt — STUB (spike)

This file exists to prove one thing during the plumbing spike: a **reusable
workflow** running from `cad-console` can check out `cad` (the engine) and read
a prompt from it — i.e. the prompt library lives in the framework repo, not in
the app repo.

If you can see this text in the `spike-saba` run log, **Q2 (engine-prompt
access) is green.**

The real SA/BA prompt (copied from the reference implementation during full
extraction) replaces this stub once the plumbing is validated.
